Robot Vacuum
Episodes (8)
Data Pipeline
How a robot vacuum run goes from simulation config to a reviewable 3D episode.
1
Episode Config
8 episodes defined with failure class and pre-seeded reviewer decision (approve / reject / flag).
{"id":"ep_002","failure_class":"stuck_corner", "pre_decision":"flag"}
↓
2
Waypoint Generation
Random walk through a 6×6 m room — 5–8 intermediate waypoints connecting start to goal.
waypoints = [[0.45,−0.63],[1.23,0.89], [2.10,0.41],[2.87,1.14]]
↓
3
Pure Pursuit Simulation @ 50 Hz
Differential-drive controller steers toward each waypoint. Produces an [x, y, θ] state per timestep.
states[i] = [x + v·cos(θ)·dt, y + v·sin(θ)·dt, θ + ω·dt]
↓
4
Failure Injection
At failure_step, the failure mode kicks in — oscillation, drift, stall, or e-stop — altering v and ω.
if step ≥ failure_step: ω += 2.4 × sin(t × 1.1) # stuck
↓
5
FastAPI — Episode JSON
Server exposes each episode over REST. The browser fetches on episode select.
GET /api/episodes/ep_002 → {states, v_linear, v_angular, waypoints, failure_step, ...}
↓
6
Three.js 3D Render
Each frame: robot disc positioned at states[step], speed-colored trail drawn, LED ring red on failure.
robot.position.set(x, 0, y) robot.rotation.y = −θ ledRing.color = failure ? red : green
↓
7
Human Review
Reviewer watches playback, selects Approve / Reject / Flag (with reason label), submits decision.
POST /api/review/ep_002 {decision:"flag",notes:"[Stuck/Trapped]..."}
