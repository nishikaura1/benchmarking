# Haptal Human-in-the-Loop Robot Trajectory Reviewer

[Haptal](https://haptal.ai) builds failure intelligence for robot training data. We automatically detect and label failure modes in raw robot trajectories so your training pipeline gets clean, physics-grounded failure labels instead of unlabeled noise. Our public benchmark is at [HaptalAI/robotics-failure-benchmark](https://huggingface.co/datasets/HaptalAI/robotics-failure-benchmark).

---

## What This Tool Is

This is a human-in-the-loop annotation tool. You watch robot trajectories animate in 3D — the robot moves through a room, the path draws behind it, and failures are highlighted in red. You decide: does this episode look like a real failure or a false positive? Pass or reject it. When you're done, download a clean curated CSV ready to plug into your training pipeline.

---

## How to Use It

1. Select a dataset from the dropdown
2. Enter an episode number (or use Prev / Next to browse)
3. Click **Load Episode** — the robot starts animating immediately
4. Watch the trajectory. Green path = normal. Red = failure detected at that timestep
5. Use the scrubber to jump to any step. Use 0.5×/1×/2× to control playback speed
6. Click **✓ Pass** if the failure label looks correct, **✗ Reject** if it's wrong or a false positive
7. Add an optional rejection reason before rejecting
8. After 5 episodes reviewed, the download section appears
9. Enter your email and download your curated dataset as CSV

---

## What the Downloaded Dataset Contains

| Column | Description |
|--------|-------------|
| `episode_id` | The episode identifier from the source dataset |
| `dataset` | Which dataset this episode came from |
| `human_label` | Your decision: `pass` or `reject` |
| `rejection_reason` | Your optional rejection note (empty for passed episodes) |
| `reviewer_session_id` | Unique ID for your review session |
| `review_timestamp` | UTC timestamp when you made the decision |

## How to Use the Downloaded Dataset in Your Training Pipeline

```python
import pandas as pd
from datasets import Dataset

# Load your curated dataset
df = pd.read_csv('haptal_curated_dataset.csv')

# Filter passed episodes only
clean = df[df['human_label'] == 'pass']

# Convert to HuggingFace dataset
dataset = Dataset.from_pandas(clean)
print(f'Clean episodes ready for training: {len(dataset)}')
```

## About the AMR Dataset

[HaptalAI/amr-navigation-failure-dataset](https://huggingface.co/datasets/HaptalAI/amr-navigation-failure-dataset) is a synthetic autonomous mobile robot navigation dataset with 500 episodes across 5 failure classes: nominal, path_deviation, velocity_spike, stuck, and overcorrect. Each episode is a step-level trajectory with x/y position, heading, velocity, and labeled failure timestep. Generated using physics-based simulation of a differential-drive robot in a 10×10 metre room.

---

## About Episode IDs

For the AMR dataset, episode IDs are sequential synthetic counters: `{failure_class}_{index:04d}`. The index simply counts how many episodes of that class were generated — `stuck_0012` is the 13th stuck episode. It is not an index into any external dataset.

For LeRobot datasets, the episode number corresponds directly to `episode_index` in the source dataset, starting at 0.

This tool provides 3D trajectory visualization and basic heuristic anomaly detection only. The failure labels shown are generated by simple velocity anomaly detection — not our full pipeline.
